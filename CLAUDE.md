# Deck Box — notes for Claude

Parametric MTG EDH deck box. Box body in OpenSCAD, Voronoi scale pattern precomputed in Python.

## Files

- `deckbox.scad` — main geometry. Rounded box, cavity, decorative windows, snap-fit bumps.
- `seam.py` — Voronoi preprocessor. Emits `seam.scad` with `base_teeth()` + `lid_teeth()` polyhedra.
- `seam.scad` — **auto-generated**, do not edit by hand. Regenerated by `seam.py`.
- `build.sh` — `python seam.py && openscad … base.stl lid.stl assembly_*.png`. ~3s end-to-end. Pass `--step` to additionally emit `assembly.step` (slow, see STEP export below).
- `csg_to_step.py` — FreeCAD script that converts `assembly.csg` to `assembly.step`. Run via `freecadcmd`, not the system python.

## Stack choice

OpenSCAD with `--backend=Manifold`. **Do not switch to Build123d/OCCT** — the same Voronoi geometry took 60+ minutes there with invalid topology output. Manifold does it in ~2s. See the `scripting-cad-choice` memory for the full story.

## Dimensions kept in sync between two files

`CAV_W`, `CAV_D`, `CAV_H`, `CAV_R`, `WIDE_WALL_T`, `NARROW_WALL_T`, `FLOOR_T`, `CEILING_T`, `SEAM_Z`, `VERTICAL_GAP` appear in **both** `deckbox.scad` and `seam.py`. When changing any of these, edit both files or the scale geometry will misalign from the wall.

## Gotchas

### Shapely orientation is not guaranteed

`Polygon.intersection(bounds)` returns rings that may be **clockwise**, not CCW. The custom `shrink_voronoi_edges_only()` computes inward normals from edge direction, which only works for CCW rings — wrong orientation makes "inward" point outward, so scales grow instead of shrinking and merge into a continuous offset surface (no visible cell boundaries on flat faces). **Always call `orient(poly, sign=1.0)` before per-edge offset work.**

### Voronoi convex-hull rejection

scipy.spatial.Voronoi returns `-1` vertices for cells of seeds on the convex hull. For a narrow z band (e.g. the 22mm base region), **~25% of seeds land on the z-extremes and produce unbounded regions** that get rejected. `voronoi_with_tiling()` adds phantom seeds at `z = CELLS_Z_MIN - 5` and `z = CELLS_Z_MAX + 5` to close the hull. This is required, not optional — without phantoms the base looks visibly sparser than the lid even at the same target density.

### `buffer(-gap)` shrinks clip-line edges too

A uniform `polygon.buffer(-SCALE_GAP, join_style=2)` is the obvious way to inset for inter-cell gaps, but it pulls *every* edge inward — including edges that came from clipping against the seam (`z = SEAM_Z`, `z = LID_BOTTOM_Z`). That widens the visible seam gap by `2 × SCALE_GAP` and the base loses scales more visibly than the lid because it's a shorter region. The current `shrink_voronoi_edges_only()` walks edges, detects which lie on the bounding box (clip lines), and offsets only the Voronoi-ridge edges. **Don't replace it with a plain `buffer()` call.**

### Base/lid Voronoi tessellations are separate

Cell seeds are sampled independently for base (z ∈ [BASE_BOT_Z, SEAM_Z]) and lid (z ∈ [LID_BOTTOM_Z, LID_TOP_Z]) with counts proportional to each region's area at `CELL_DENSITY` cells/mm². The two zones never share Voronoi ridges — they live in disjoint z ranges separated by `VERTICAL_GAP`. This makes density-matching trivial and means the seam doesn't have any cross-zone cells to align.

### "NotManifold" warnings during assembly preview are benign

`openscad` prints `WARNING: PolySet -> Manifold conversion failed: NotManifold / Trying to repair and reconstruct mesh..` during the colored assembly preview render. Each step still ends with `Status: NoError` and a manifold result. The STL outputs are always clean — only the multicolor preview triggers the warnings, and they auto-repair.

### Snap-fit: bumps on **long sides only**

Two bumps at `y = ±NARROW_D/2` (the long-axis sides). Release motion is "pinch the short sides" — squeezing the unbumped short sides bows the long sides outward and pops the bumps free of their cavities. Adding bumps to all four sides removes the release mechanism and makes the lid much harder to open.

### Cards lie flat — "landscape" cavity

`CAV_W` (X) holds card *height* 91.3, `CAV_D` (Y) holds card *width* 68.3, `CAV_H` (Z) is the *stack* height. The mapping is not obvious from the variable names — they're cavity dimensions, not card dimensions.

### STEP export goes via FreeCAD CSG import — not OpenSCAD

OpenSCAD has no BRep kernel and can't export STEP at all. The path used by `build.sh --step` is:
`OpenSCAD → assembly.csg (CSG tree) → freecadcmd csg_to_step.py → assembly.step`.
FreeCAD's CSG importer rebuilds the box body's primitives as real BRep (planes, cylinders, spheres), but the 701 Voronoi `polyhedron(...)` calls pass through as faceted BReps — every triangle becomes a planar face. That's why `assembly.step` is ~43 MB and conversion takes ~9 min. There is no obvious way to make this faster without giving up the per-cell polyhedron approach.

Two `freecadcmd` invocation gotchas: (a) it does **not** set `__name__ == "__main__"`, so any `if __name__ == "__main__":` guard silently skips the script body; (b) `print()` output is interleaved with FreeCAD's own progress spam, use `flush=True` to keep timing visible.

### Dead modules / legacy stubs

- `module voronoi_cutters() { }` in `seam.scad` is an empty stub kept for back-compat. The old design had cell-boundary grooves carved into a seam band; that's gone, replaced with the protruding-scales design. The stub does nothing — leave it alone unless you also remove every reference.
- The "split STL for multi-material" outputs (`base_main`, `base_scales`, `lid_main`, `lid_scales`) were removed in favor of combined parts only. Don't re-add unless explicitly asked.

## Print settings

- FDM, PLA or PETG.
- H_TOL = 0.3 mm per side (0.6 mm diametral). Loosen to 0.4 if your printer runs tight.
- Snap-fit bump interference is `LOCK_BUMP_R − H_TOL = 0.6 mm` of wall flex per bump during slide-down. Firm on a 1.86 mm wall in PLA/PETG; reduce `LOCK_BUMP_R` if it's too stiff to open.
- Print orientation: base open-side-up, lid open-cavity-up. Both have flat undersides for first-layer adhesion.

## Quick reference

- Total build: `bash build.sh` (regen + STLs + 2 previews, ~3s)
- Change cell density: `CELL_DENSITY` in seam.py (cells/mm²)
- Change scale prominence: `SCALE_PROTRUSION` (outward), `SCALE_GAP` (between-cell gap), `SCALE_OVERLAP` (into-wall attachment)
- Change snap stiffness: `LOCK_BUMP_R` (smaller = easier to open)
